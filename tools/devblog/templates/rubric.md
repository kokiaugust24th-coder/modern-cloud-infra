# 品質ルーブリック

自己批評段階(devblog generate の第3段)は、この観点で下書きを自己採点し、基準を満たさない
項目を修正する。採点結果は PR 本文に参考情報として添付する(リンター不合格の上書きには使わない)。

| 観点 | 基準 |
|---|---|
| 読者価値 | 読み終えたとき、読者が「何が変わったか」「なぜそう判断したか」を説明できる内容になっているか |
| 構成 | 導入 → 本文(テーマ別) → まとめの流れが破綻していないか。テーマの粒度が揃っているか |
| タイトル | 内容を正確に表し、誇張がないか。クリックベイト的な煽り文句がないか |
| 具体性 | 「色々改善した」のような曖昧な記述がなく、ダイジェストの事実に基づく具体的な記述になっているか |
| 日本語品質 | 誤字脱字・不自然な表現がないか(textlint の指摘とは別に、文章としての自然さを確認する) |
| 事実接地 | ダイジェストに存在しない実装・成果・数値が書かれていないか |

各観点を 3 段階(要修正 / 許容 / 良好)で自己採点し、「要修正」があれば該当箇所を修正してから
最終稿として出力する。
