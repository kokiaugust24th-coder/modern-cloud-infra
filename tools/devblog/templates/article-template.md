# 記事構成テンプレート

1. **タイトル**: 期間中の主要な変更点を一言で表す。誇張しない(「完璧」「革命的」等は禁止)
2. **導入(2〜3文)**: 対象期間・何をした週かを一言で。読者が続きを読む価値を判断できる情報を先出しする
3. **本文**:
   - 見出し(`##`)ごとに 1 つの変更テーマ(1 OpenSpec チェンジ、または関連コミット群)を扱う
   - 各テーマは「なぜ着手したか → 何をしたか → 何を学んだか/どう判断したか」の順で書く
   - 構造的な変更(複数コンポーネントにまたがるもの)には mermaid 図を添える
   - 開発メモに記載がある場合は「学び」セクションに反映する
4. **まとめ(2〜3文)**: 次にやること・オープンな課題があれば触れる
5. **情報源(機械生成、編集しない)**: コミット範囲・OpenSpec チェンジ名のリンクカード
6. **フッター(機械生成、編集しない)**: ライセンス表記

## 禁止事項

- ダイジェストに含まれない実装・数値・成果を書かない(事実接地の原則)
- 「〜だと思われます」等の曖昧表現より、ダイジェストの事実を淡々と書く
- 画像ファイルへの新規参照を含めない(画像は PR レビュー時に人間が追加する)
