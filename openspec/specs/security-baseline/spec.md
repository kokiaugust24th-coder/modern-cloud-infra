## Requirements
### Requirement: 最小権限の IAM
人およびワークロードのアイデンティティには、職務・機能に必要な最小限の権限のみが付与されなければならない (MUST)。ワークロードは長期的な静的クレデンシャルではなく、短期の一時クレデンシャル（ロール引き受け／Workload Identity）を使用しなければならない (MUST)。

#### Scenario: 権限外操作の拒否
- **WHEN** ワークロードが自身のロールに許可されていないリソースへアクセスしたとき
- **THEN** アクセスは拒否され、監査ログに記録される

#### Scenario: 静的キーの排除
- **WHEN** ワークロードがクラウド API を呼び出すとき
- **THEN** 実行環境に紐づく一時クレデンシャルが自動取得され、コードや設定に静的アクセスキーは存在しない

### Requirement: 保存時・転送時の暗号化
すべてのデータストアとストレージは保存時に暗号化されなければならない (MUST)。サービス間および外部との通信は TLS で暗号化されなければならない (MUST)。

#### Scenario: 非暗号化リソースの防止
- **WHEN** 暗号化無効のストレージやデータベースが作成されようとしたとき
- **THEN** ポリシーによって作成が拒否される

### Requirement: シークレット管理
API キー・パスワード等のシークレットは専用のシークレットストアで管理されなければならない (MUST)。ソースコード・IaC 定義・コンテナイメージにシークレットを平文で含めてはならない (MUST NOT)。シークレットはローテーション可能でなければならない (SHALL)。

#### Scenario: 実行時のシークレット取得
- **WHEN** アプリケーションがシークレットを必要とするとき
- **THEN** 実行時にシークレットストアから取得され、取得はアクセスログに記録される

#### Scenario: コミット時の検知
- **WHEN** シークレットらしき文字列を含む変更がプッシュされたとき
- **THEN** シークレットスキャンが検知し、マージがブロックされる

### Requirement: 監査ログと継続的なセキュリティ評価
クラウド API の操作ログ（監査証跡）は全環境で有効化され、改ざん不能な形で保管されなければならない (MUST)。設定ミス・脆弱性の継続的スキャンが実行され、重大な検出は通知されなければならない (MUST)。

#### Scenario: 侵害調査
- **WHEN** セキュリティインシデントが疑われるとき
- **THEN** 誰が・いつ・どのリソースに・何をしたかを監査ログから再構成できる
