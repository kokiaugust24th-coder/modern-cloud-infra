# dev-activity-collection

## ADDED Requirements

### Requirement: 期間指定による開発情報の収集
システムは、指定された期間（開始日時・終了日時）を対象に、Git コミット履歴および OpenSpec チェンジ成果物（proposal / design / tasks の本文と進捗）を収集し、機械可読な開発ダイジェスト（JSON）として出力しなければならない（MUST）。

#### Scenario: 週次期間のダイジェスト生成
- **WHEN** 過去 7 日間を対象として収集コマンドを実行する
- **THEN** 対象期間内のコミット（ハッシュ・メッセージ・変更ファイル名・追加/削除行数）と、期間内に更新された OpenSpec チェンジの一覧を含むダイジェスト JSON が出力される

#### Scenario: 対象期間に活動がない場合
- **WHEN** 対象期間内にコミットも OpenSpec チェンジの更新も存在しない
- **THEN** システムは空のダイジェストを出力し、後段（記事生成）を実行しない終了コードを返す

### Requirement: ソースコード本文の収集禁止
収集処理は、ソースコードの本文および diff 本文をダイジェストに含めてはならない（MUST NOT）。収集対象はコミットメタデータ（ハッシュ、メッセージ、変更ファイルパス、統計）と OpenSpec ドキュメント本文に限定しなければならない（MUST）。

#### Scenario: diff 本文の除外
- **WHEN** 収集対象期間にコード変更を含むコミットが存在する
- **THEN** ダイジェストには変更ファイルのパスと行数統計のみが含まれ、コードの中身・diff 本文は含まれない

#### Scenario: 除外パスの適用
- **WHEN** `.env*`・`secrets/` 等の除外パターンに一致するファイルがコミットに含まれる
- **THEN** それらのファイルパスはダイジェストから除外される

### Requirement: 開発メモの収集
システムは、開発者が任意で書き溜める開発メモファイル（`devlog/` 配下の日付付き Markdown。学び・ハマりどころ・参照 URL・埋め込み候補の URL を含む）を収集対象に含めなければならない（MUST）。開発メモは任意の素材であり、存在しない場合でも収集・生成は成立しなければならない（MUST）。

#### Scenario: メモが存在する週の収集
- **WHEN** 対象期間内の日付を持つ開発メモが `devlog/` に存在する状態で収集を実行する
- **THEN** ダイジェストにメモ本文と記載された参照 URL・埋め込み候補 URL が含まれる

#### Scenario: メモが存在しない週の収集
- **WHEN** 対象期間内に開発メモが存在しない状態で収集を実行する
- **THEN** 収集はエラーにならず、コミットと OpenSpec 情報のみのダイジェストが出力される

### Requirement: 収集済み期間の重複防止
収集処理は公開履歴台帳を参照し、既に記事化・公開済みの期間・情報源を再収集の既定対象から除外しなければならない（MUST）。ただし明示的なオプション指定による再収集は許可される（SHALL）。

#### Scenario: 公開済み期間のスキップ
- **WHEN** 前回の公開履歴に記録済みの期間を含む範囲で収集を実行する
- **THEN** 既定では公開済み期間の後（未公開分）のみが収集対象となる

#### Scenario: 強制再収集
- **WHEN** 再収集オプションを明示指定して収集を実行する
- **THEN** 公開履歴に関わらず指定期間全体が収集される
