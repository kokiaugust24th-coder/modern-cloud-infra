# publication-content-safety

## ADDED Requirements

### Requirement: 公開前の決定的な機密情報検査
システムは、公開対象コンテンツに対し決定的なパターン検査（シークレットスキャナ + プロジェクト固有パターン定義）を実行し、機密情報の検出時は公開フローを停止しなければならない（MUST）。検査対象パターンには少なくとも API キー・トークン形式、認証情報、メールアドレス、内部ホスト名・URL を含めなければならない（MUST）。

#### Scenario: 機密情報の検出によるブロック
- **WHEN** 記事下書きに API キー形式の文字列が含まれた状態で検査を実行する
- **THEN** 検査は失敗（非ゼロ終了コード）となり、検出箇所（行番号・パターン種別）がレポートされ、公開フローは先へ進まない

#### Scenario: 検査合格
- **WHEN** 機密パターンに一致しない記事下書きに対して検査を実行する
- **THEN** 検査は合格し、合格した内容のハッシュを含む検査結果が記録される

### Requirement: 非決定的判定の位置づけ
LLM 等の非決定的コンポーネントによる公開可否の判定は、警告の付与に用いてよい（MAY に相当する運用を許可する）が、決定的検査によるブロックを解除する根拠として使用してはならない（MUST NOT）。ブロックの解除は人間の明示的な操作によってのみ行われなければならない（MUST）。

#### Scenario: 誤検出の人間による解除
- **WHEN** 決定的検査が誤検出（例: ドキュメント上のダミーキー）によりブロックし、人間が該当検出を確認して許可リストに追加する
- **THEN** 再検査は合格し、許可リストへの追加が理由・追加者とともに記録される

### Requirement: 検査の網羅性の担保
公開フローは、検査に合格していないコンテンツ、または検査合格後に変更されたコンテンツを公開対象にしてはならない（MUST NOT）。

#### Scenario: 検査後の改変検知
- **WHEN** 検査合格後に記事ファイルが変更され、公開ステップが実行される
- **THEN** 内容ハッシュの不一致により公開は拒否され、再検査が要求される
