# cost-management

## ADDED Requirements

### Requirement: 予算アラートの事前設定
課金が発生しうる環境（クラウドアカウント・PaaS プラン）には、リソース作成に先立って月次予算と閾値アラート（予測ベースを含む）が設定されなければならない (MUST)。

#### Scenario: 予算超過見込みの通知
- **WHEN** 当月のコスト予測が設定予算の閾値（例: 80%）を超過したとき
- **THEN** 責任者へ通知が送られ、超過要因のサービス別内訳が確認できる

#### Scenario: アラートなしの課金環境の防止
- **WHEN** 予算アラート未設定のアカウントで課金リソースが作成されようとしたとき
- **THEN** ポリシーチェックまたはセットアップ手順によりアラート設定が先に要求される

### Requirement: コスト配分タグの必須化
課金対象のクラウドリソースには、環境（env）・サービス（service）・所有者（owner）を示すタグが付与されなければならない (MUST)。タグのないリソース作成は IaC のポリシーチェックで拒否されなければならない (MUST)。

#### Scenario: タグ欠落のブロック
- **WHEN** 必須タグを欠いたリソース定義がプルリクエストに含まれたとき
- **THEN** CI のポリシーチェックが失敗し、マージがブロックされる

### Requirement: 環境・サービス別のコスト可視化
コストは環境別・サービス別に集計され、月次でレビューされなければならない (MUST)。売上またはユーザー数に対するインフラ費率が追跡されなければならない (SHALL)。

#### Scenario: コスト異常の特定
- **WHEN** 月次コストが前月比で著しく増加したとき
- **THEN** タグ集計によりどの環境・サービスが増加要因かを特定できる

### Requirement: 無料枠・クレジットの消化監視
無料枠およびスタートアップクレジットの残高と消化ペースは監視されなければならない (MUST)。クレジットの消化予測が 90 日を切った時点で通知が発報され、フェーズ移行またはコスト最適化のレビューが開始されなければならない (MUST)。

#### Scenario: クレジット枯渇前の警告
- **WHEN** 現在の消化ペースでクレジットが 90 日以内に枯渇すると予測されたとき
- **THEN** 通知が発報され、Savings Plans・Spot 化・構成縮小などの最適化レビューが起票される

### Requirement: クレジット対象外費用の管理
クラウドクレジットで相殺されないサードパーティ費用（PaaS の有料プラン、外部 SaaS 等）は、実費として予算に計上され、無料プラン上限への接近が監視されなければならない (MUST)。有料プラン移行が必要になる見込みが生じた時点で、実費影響の見積りと代替手段（クレジット適用可能なサービスへの移行等）の評価が起票されなければならない (MUST)。

#### Scenario: サードパーティ有料化の事前評価
- **WHEN** サードパーティサービス（例: Supabase Auth）の無料プラン上限の使用率が 70% に到達したとき
- **THEN** 通知が発報され、有料プラン継続とクレジット適用可能な代替（例: Cognito 移行）の比較評価が起票される

### Requirement: フェーズ移行トリガーの監視
design.md に定義されたフェーズ移行トリガー（PaaS プラン上限の 70% 到達、クレジット枯渇 90 日前 等）に用いる指標は継続的に計測されなければならない (MUST)。

#### Scenario: PaaS 上限接近の検知
- **WHEN** フェーズ0 の PaaS 無料プランの上限（帯域・DB 容量・同時接続のいずれか）の使用率が 70% に到達したとき
- **THEN** 通知が発報され、フェーズ1（AWS 移行）の計画が開始される

### Requirement: 非本番環境のコスト抑制
非本番環境（dev / staging）は、業務時間外の自動停止またはゼロスケールにより、稼働コストが抑制されなければならない (SHALL)。

#### Scenario: 夜間の自動停止
- **WHEN** 定義された業務時間外になったとき
- **THEN** 非本番環境のコンピュートとデータベースは自動停止し、業務開始前に自動再開する
