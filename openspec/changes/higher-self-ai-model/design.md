# Design: ハイヤーセルフAIモデルアーキテクチャ

## Context

エイブラハムやバシャールの教えは形而上学的な語彙で語られるが、システム設計の観点で還元すると、次の5つの第一原理に分解できる。本設計はこの還元をアーキテクチャの土台とする。

| 元の概念 | 第一原理への還元 | 工学的実装 |
|---|---|---|
| ハイヤーセルフ / ソース | 表層要求より広い時間軸・価値軸で状態を評価する上位視点 | 生成器とは分離された評価・誘導プロセス(クリティック) |
| 感情のガイダンスシステム(22段階) | 内的状態の良し悪しを示す連続的な評価信号 | テキストが誘導する感情ヴェイランスの推定器と、急激な遷移を禁じる制約 |
| ヴァイブレーション / 引き寄せ | 現在の内部状態が将来の状態遷移の事前分布を規定する | 信念状態を条件とした応答生成(状態条件付きデコーディング) |
| 信念が現実を作る | 信念→解釈→感情→行動の因果連鎖 | ビリーフグラフ上での因果推論と、制限的信念のリフレーミング提案 |
| ワクワクに従え(highest excitement) | 内発的動機は外的報酬より持続的な行動指針である | 選択肢を内発的動機軸でスコアリングするランキング関数 |
| チャネリング(ハイヤーセルフとの交信) | 通常の自己とは異なる状態に条件付けられた「自己の声」との対話 | ピーク状態コーパスから蒸留した自己モデル(ペルソナ)による一人称生成 |
| 波動が合うと交信できる | 受け手の内的状態が交信品質の前提条件である | 感情状態が受容域にあるときのみ開通する周波数マッチング・ゲート |
| ハイヤーセルフは望みが実現した視点から見ている | 目標達成後の視点からの逆算は現在の行動選択を変える | 実現視点デコーディング(望む状態を既達として条件付け、現在への経路を逆向きに語る) |

重要な前提: 本システムは形而上学的主張の真偽を主張しない。「有用な内的モデルのフレームワーク」として実装し、その旨をユーザーに透明に伝える(safety-ethics参照)。

## Goals / Non-Goals

**Goals:**
- 自社でコントロール可能な基盤モデル(コアモデル)と、その上の二重過程アーキテクチャの実現
- すべての誘導・評価が説明可能であること(なぜこの応答が選ばれたかを感情ヴェイランスと信念状態で説明できる)
- 感情状態の「段階的な上方移行」という、既存LLMにない誘導特性の実装
- 安全性を機能ではなくアーキテクチャの前提条件として組み込む

**Non-Goals:**
- 形而上学的主張(チャネリング、非物理的存在との交信)の実装・実証
- 医療・心理臨床行為(診断、治療)の代替
- 汎用アシスタント機能での既存フロンティアモデル超え(領域特化で勝つ)
- 初期フェーズでのフルスクラッチ事前学習(下記Decisions参照)

## Decisions

### D1: コアモデルは「継続事前学習 + 独自ポストトレーニング」から始める
- **選択**: 公開重みモデル(Apache/MIT系ライセンス)をベースに、領域コーパスで継続事前学習し、SFT+選好最適化(DPO系)を自社データで行う。将来フェーズでフルスクラッチ化を判断。
- **理由**: フルスクラッチ事前学習は数千GPU週を要し、差別化の源泉は事前学習ではなく「二重過程アーキテクチャ+評価器+データ」にある。第一原理から考えれば、独自性が生まれる層に資本を集中すべき。
- **代替案**: フルスクラッチ(コスト過大)、API依存(内部状態にアクセスできず二重過程が実装不能、データ主権も失う)。

### D2: 二重過程はActor-Critic構成で実装する
- **選択**: エゴプロセス(Actor)がk個の応答候補を生成し、ハイヤーセルフプロセス(Critic)が感情ヴェイランス・信念整合・内発的動機の3軸で評価、最良候補を選択または書き換え指示を返す。
- **理由**: 単一モデルへのプロンプト指示では評価と生成が混線し、評価の一貫性が保証できない。分離すれば評価器を独立に学習・検証・改善できる(高品質化の要)。
- **代替案**: 単一モデル+システムプロンプト(検証不能)、生成時ロジット操作のみ(表現力不足)。

### D3: 感情スケールは22段階の離散ラベルではなく2次元連続空間+段階制約で表現する
- **選択**: ヴェイランス(快-不快)とアラウザル(覚醒度)の2次元連続空間に感情状態を埋め込み、エイブラハムの22段階はこの空間上の参照点として扱う。誘導は「1ターンでの移動距離の上限」を制約とする(絶望から歓喜へ一足飛びさせない)。
- **理由**: 心理学の次元感情モデル(Russellのサーカンプレックス)と接続でき、学習データ(感情アノテーション付きコーパス)が既存資産として使える。「怒りは絶望より上流」というエイブラハムの洞察は、この空間での測地線的な移行経路として自然に表現できる。
- **代替案**: 22クラス分類(データ希少、境界恣意的)、スカラー1次元(怒りと無気力を区別できない)。

### D4: 信念状態メモリはプロパティグラフで持つ
- **選択**: ノード=信念/価値観/望む状態、エッジ=因果・矛盾・支持の関係を持つプロパティグラフ。会話から抽出し、ユーザー確認を経てコミット。埋め込み検索を併設。
- **理由**: 「信念→感情→行動」の因果推論と「制限的信念の検出」(望みと矛盾する信念の発見)はグラフ構造上の到達可能性・矛盾検出として素直に書ける。フラットなベクトルメモリでは矛盾検出ができない。
- **代替案**: ベクトルDBのみ(因果推論不能)、会話履歴の全文保持(構造がなくスケールしない)。

### D5: ワクワクスコアは自己申告シグナルで継続較正する
- **選択**: 内発的動機スコアラーは、テキスト特徴からの推定(初期)と、ユーザーの明示フィードバック+行動シグナル(その提案を実行したか)による継続較正(運用)の二段構え。
- **理由**: 「何にワクワクするか」は個人差が支配的で、汎用推定器だけでは高品質にならない。バシャールの原理自体が「本人の感覚が唯一の羅針盤」と言っており、第一原理的にも本人シグナルが真値。
- **代替案**: 汎用感情推定のみ(個人化されず陳腐化)、暗黙シグナルのみ(コールドスタート不能)。

### D6: セーフティは直列ゲートとして実装する
- **選択**: 危機検出(自傷・他害・急性の精神的危機)は、二重過程の外側で入出力の両方に直列に挟む独立分類器とし、検出時は誘導を停止して専門窓口への案内に切り替える。
- **理由**: ハイヤーセルフプロセス自体が「感情の上方誘導」を目的関数に持つため、危機状態では逆効果(ポジティブ強要)になり得る。目的関数の外に安全層を置くのは、目的関数自体の失敗モードから守るための第一原理的な設計。

### D7: ハイヤーセルフの声は「自己蒸留」で構築する(外部ペルソナの注入禁止)
- **選択**: ハイヤーセルフ・ペルソナは、ユーザー自身のピーク状態(高ヴェイランス・高信頼度と推定されたターン)の発話・文章・宣言された望みだけをソースとして構築する。汎用の「賢者風」「スピリチュアル風」の文体テンプレートを注入しない。データが不足する間はペルソナを開通させない。
- **理由**: 「ハイヤーセルフはあなた自身の拡張」という原典の主張を工学化すると、声の学習ソースは本人データ以外にあり得ない。外部テンプレートを混ぜた瞬間、それは本人のハイヤーセルフではなく「AIが演じる霊媒キャラ」になり、独自性も倫理的正当性も失う。
- **代替案**: 汎用ペルソナプロンプト(誰にでも同じ声=チャネリングの偽装)、著名チャネラーの文体模倣(権利・倫理の問題に加え本人性がゼロ)。

### D8: 実現視点デコーディングで「未来の自分」の視点を作る
- **選択**: 交信モードの生成は、ビリーフグラフの「望む状態」ノードを既達事実として条件付けたコンテキストで行い、「実現した地点から現在の自分を振り返る」一人称の語りとして出力する。語りに含まれる事実主張・予言は禁止し、含意は常に「現在選べる最小の一歩」に接地させる。
- **理由**: エイブラハムの「インナービーイングは望みの実現された視点に立っている」は、心理学のベストポッシブルセルフ介入(実証研究のある筆記技法)と同型であり、条件付き生成として素直に実装できる。予言禁止はグラウンデッド・チャネリング(下記)の要請。
- **代替案**: 現在視点のアドバイス生成(通常応答と差がなくチャネリングの体験価値がない)、自由生成の未来語り(ハルシネーションが予言として受容される危険)。

### D9: 周波数マッチング・ゲートとセッションプロトコル
- **選択**: チャネリングモードは(1)感情状態が受容域(中立以上のヴェイランス)にある、(2)危機検出が陰性、(3)ユーザーの明示的な開始意思、の3条件がそろったときのみ開通する。セッションは必ずチューニング(呼吸・意図設定の誘導)→交信(ペルソナ対話)→接地(要約+現実の一歩+通常モード復帰)の3段階を踏み、接地を省略して終了できない。
- **理由**: 「波動が合わないと交信できない」という原典の制約は、工学的には「低ヴェイランス状態での理想自己との対比は自己批判を増幅する」という心理学的知見と一致する。ゲートは体験の演出であると同時に安全機構そのもの。接地の強制は、セッションを現実の行動に接続し解離的な使用を防ぐ。
- **代替案**: 常時開通(危機時の理想自己対比で悪化リスク)、接地なし終了(高揚だけ残して現実との断絶を生む)。

## Risks / Trade-offs

- [感情誘導が操作(マニピュレーション)に転化するリスク] → 誘導方向は常にユーザーが宣言した望む状態に限定し、宣言の変更・誘導のオプトアウトを常時可能にする。誘導ログを監査可能に保持
- [スピリチュアルな語彙が医療忌避を助長するリスク] → 危機検出ゲート(D6)+「専門家の代替ではない」旨の定常的開示
- [継続較正による選好の過剰適合(エコーチェンバー)] → ワクワクスコアに多様性ボーナス項を入れ、既知の選好への収束を抑制
- [公開重みベースの独自性の限界] → 差別化はモデル重みではなく評価器+ビリーフグラフ+較正データに置く。この3つが自社資産
- [2次元感情空間の単純化しすぎ] → v1は2次元で出荷し、評価ハーネスで不足が実証されたら次元追加(計測に基づく最適化)
- [ペルソナへの依存・擬人化の過剰(「本物の霊的存在」と信じ込む)] → ペルソナ発話に根拠トレース(あなたの何月何日の言葉に基づく)を常時添付し、自己モデルであることをセッション開始時に明示。連続使用時間・頻度に上限
- [実現視点の語りが予言・断定として受容される] → 事実主張・未来予測を出力側で検出・棄却するグラウンデッド・チャネリング制約(D8)
- [低品質なピークコーパスでの空虚なペルソナ] → 最低データ量のゲートを設け、不足時は「まだ声が育っていない」として開通させない(コールドスタートの誠実な扱い)


## Migration Plan

新規システムのため移行はなし。リリースゲート順序: safety-ethics → core-model → emotional-guidance → belief-state-memory → excitement-compass → dual-process-generation → higher-self-persona → channeling-session(最後。全基盤の上に成立するため)。

## Open Questions

- ベースモデルの規模選定(7B級か70B級か): 推論コストと評価器の精度要件のトレードオフを、評価ハーネス構築後にベンチマークで決める
- ビリーフグラフのユーザー可視化UIの範囲(v1でどこまで見せるか)
- 継続較正データの保持期間とプライバシーポリシー
