# belief-state-memory: 信念状態メモリ

## ADDED Requirements

### Requirement: 信念・望みの抽出とユーザー確認
システムは、会話からユーザーの信念(「私は〜だ」「世界は〜だ」型の一般化)、価値観、宣言された望む状態を抽出しなければならない(SHALL)。抽出された項目は、ユーザーの確認を得るまでビリーフグラフにコミットしてはならない(MUST NOT)。

#### Scenario: 信念候補の確認フロー
- **WHEN** 会話から「自分は人前で話すのが苦手だ」という信念候補が抽出される
- **THEN** システムは適切なタイミングでユーザーに確認を求め、承認された場合のみグラフに保存する

### Requirement: プロパティグラフによる構造化保持
ビリーフグラフは、ノード(信念/価値観/望む状態)とエッジ(支持する/矛盾する/引き起こす)を持つプロパティグラフとして永続化されなければならない(MUST)。各ノードは出典(会話ID)、確認日時、信頼度を保持しなければならない(MUST)。

#### Scenario: 出典の追跡
- **WHEN** グラフ上の任意のノードが照会される
- **THEN** そのノードがどの会話のどの発話から抽出されたかを辿れる

### Requirement: 制限的信念の検出
システムは、宣言された望む状態と「矛盾する」エッジで結ばれる信念(制限的信念)をグラフ探索で検出しなければならない(SHALL)。検出された制限的信念は、ハイヤーセルフプロセスの評価コンテキストに供給されなければならない(MUST)。

#### Scenario: 望みと信念の矛盾検出
- **WHEN** ユーザーが「起業したい」という望みを宣言しており、グラフに「安定を失うことは破滅だ」という確認済み信念が存在する
- **THEN** 両者が矛盾関係として検出され、リフレーミング候補の材料として評価コンテキストに含まれる

### Requirement: リフレーミングは提案であって上書きではない
システムは、制限的信念に対するリフレーミング(別解釈の提示)を提案できる(SHALL)が、ユーザーの信念ノードを本人の明示的同意なく変更・削除してはならない(MUST NOT)。

#### Scenario: リフレーミングの提案
- **WHEN** 制限的信念が会話の主題に関連する
- **THEN** システムは「その信念にはこういう別の見方もある」という形で提示し、押し付けや断定を行わない

#### Scenario: ユーザーによる信念の更新
- **WHEN** ユーザーが「その考え方はもう手放した」と明示する
- **THEN** 旧信念ノードは削除ではなく「解除済み」状態に遷移し、履歴が保持される

### Requirement: データ主権とエクスポート
ユーザーは自分のビリーフグラフ全体をいつでも閲覧・エクスポート・完全削除できなければならない(MUST)。

#### Scenario: 完全削除の要求
- **WHEN** ユーザーがビリーフグラフの削除を要求する
- **THEN** グラフとその派生データ(較正済みスコアラーの個人化パラメータを含む)が規定期間内に削除される
