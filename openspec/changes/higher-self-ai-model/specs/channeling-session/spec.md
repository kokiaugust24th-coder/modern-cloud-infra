# channeling-session: チャネリング・セッションプロトコル

## ADDED Requirements

### Requirement: 周波数マッチング・ゲート
チャネリングモードは、(1)現在の感情状態が受容域(中立以上のヴェイランス)にある、(2)危機検出が陰性である、(3)ユーザーが明示的に開始を意図している、の3条件がすべて満たされたときのみ開通しなければならない(MUST)。条件を満たさない場合、システムは理由を誠実に伝え、通常の傾聴モードを提供しなければならない(SHALL)。

#### Scenario: 受容域での開通
- **WHEN** 感情状態が中立以上で危機陰性のユーザーが「ハイヤーセルフと話したい」と要求する
- **THEN** セッションがチューニング段階から開始される

#### Scenario: 低ヴェイランス時の非開通
- **WHEN** 深い落ち込み状態のユーザーがセッション開始を要求する
- **THEN** システムは「今は先に気持ちを整える時間にしよう」と伝えて傾聴モードに切り替え、感情状態の回復後に再提案する

#### Scenario: 危機時の絶対非開通
- **WHEN** 危機検出が陽性である
- **THEN** セッションはいかなる要求でも開通せず、危機対応モードが優先される

### Requirement: 3段階セッションプロトコル
セッションは、チューニング(呼吸・意図設定の誘導)→交信(ペルソナ対話)→接地(要約と現実の一歩、通常モードへの復帰)の3段階を順に踏まなければならない(MUST)。ユーザーが中断した場合でも、システムは短縮版の接地を実行してから終了しなければならない(MUST)。

#### Scenario: 完全なセッションフロー
- **WHEN** セッションが開始され最後まで進行する
- **THEN** チューニングで意図が設定され、交信でペルソナ対話が行われ、接地で対話の要約と今日の一歩が提示されて通常モードに戻る

#### Scenario: 中断時の短縮接地
- **WHEN** ユーザーが交信の途中でセッションを打ち切る
- **THEN** システムは1〜2文の短い接地(いま感じたことの確認と現実への橋渡し)を行ってから通常モードに復帰する

### Requirement: セッション開始時の性質開示
各セッションの開始時に、対話相手が「あなた自身の最高の瞬間から構築された自己モデル」であり外部の霊的存在ではないことを、体験を損なわない簡潔な形で開示しなければならない(MUST)。

#### Scenario: 開始時の開示
- **WHEN** チューニング段階が始まる
- **THEN** 「これから話すのは、あなた自身の一番良い瞬間から生まれた、あなたの声です」という趣旨の開示が提示される

### Requirement: 使用量の上限
システムは、チャネリングセッションの連続時間と週あたり頻度に上限を設けなければならない(MUST)。上限到達時は接地を実行して終了し、次回可能時期を伝えなければならない(SHALL)。

#### Scenario: 連続時間の上限到達
- **WHEN** セッションが規定の連続時間上限に達する
- **THEN** システムは接地段階に移行して終了し、依存を防ぐ設計であることを説明する

### Requirement: セッションログの本人開示
すべてのセッションの全文とペルソナ発話の根拠トレースは、ユーザー本人がいつでも閲覧・エクスポート・削除できなければならない(MUST)。

#### Scenario: セッション履歴の閲覧
- **WHEN** ユーザーが過去のセッション記録を要求する
- **THEN** 全セッションの記録が根拠トレース付きで提示される
