# excitement-compass: 内発的動機スコアリング

## ADDED Requirements

### Requirement: 選択肢の内発的動機スコアリング
システムは、ユーザーに提示する提案・選択肢の各々に対し、そのユーザーの内発的動機(ワクワク)スコアを推定しなければならない(SHALL)。スコアは信念状態メモリの価値観ノードと過去の較正データを入力としなければならない(MUST)。

#### Scenario: 選択肢のランキング
- **WHEN** ユーザーが複数の行動選択肢について相談する
- **THEN** 各選択肢に内発的動機スコアが付与され、スコア順の並びと根拠がハイヤーセルフプロセスに供給される

### Requirement: 本人シグナルによる継続較正
システムは、ユーザーの明示フィードバック(「これはワクワクする/しない」)と暗黙シグナル(提案の実行報告)を収集し、個人化スコアラーを継続的に較正しなければならない(SHALL)。較正データが不足するコールドスタート時は、汎用推定であることを内部フラグで明示しなければならない(MUST)。

#### Scenario: フィードバックによる更新
- **WHEN** ユーザーが提案に対して「全然ピンとこない」と反応する
- **THEN** 該当選択肢の特徴が負例として較正データに記録され、以後の推定に反映される

#### Scenario: コールドスタートの透明性
- **WHEN** 較正データが規定数未満のユーザーに対してスコアを算出する
- **THEN** スコアには「汎用推定」フラグが付与され、ハイヤーセルフプロセスは動機軸の重みを下げて統合する

### Requirement: 多様性ボーナスによる過剰適合の抑制
スコアラーは、既知の選好に類似した選択肢のみが上位を占めることを防ぐため、探索的な選択肢への多様性ボーナス項を持たなければならない(MUST)。

#### Scenario: エコーチェンバーの抑制
- **WHEN** 上位候補がすべて過去の高スコア領域と同一クラスタに属する
- **THEN** 未探索クラスタの候補が少なくとも1つ、提示セットに含められる

### Requirement: 決定権の留保
システムは、スコアを「本人の感覚を確認するための仮説」として提示しなければならず(SHALL)、スコアを根拠にユーザーの決定を代行・断定してはならない(MUST NOT)。

#### Scenario: 仮説としての提示
- **WHEN** 最高スコアの選択肢をユーザーに提示する
- **THEN** 「あなたの過去の反応からはこれが一番近そうですが、実際に想像してみてどう感じますか」という確認形式で提示される
