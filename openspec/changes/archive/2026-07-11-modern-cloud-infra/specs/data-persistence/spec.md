# data-persistence

## ADDED Requirements

### Requirement: マネージドデータストアの利用
リレーショナルデータベース・キャッシュ・オブジェクトストレージは、マネージドサービスを使用しなければならない (MUST)。データベースはマルチAZ 構成で自動フェイルオーバーを備えなければならない (MUST)。

#### Scenario: プライマリ障害時のフェイルオーバー
- **WHEN** データベースのプライマリノードが属する AZ で障害が発生したとき
- **THEN** スタンバイへ自動フェイルオーバーし、アプリケーションは再接続のみでサービスを継続できる

### Requirement: 自動バックアップとポイントインタイムリカバリ
データベースは自動バックアップとポイントインタイムリカバリ（PITR）を有効化しなければならない (MUST)。バックアップの保持期間は環境ごとに定義され、本番環境では 30 日以上でなければならない (SHALL)。

#### Scenario: 誤操作からの復旧
- **WHEN** 誤ったデータ削除が発生し、発生時刻が特定できたとき
- **THEN** その直前の時点へデータベースを復元できる

### Requirement: リストアの定期検証
バックアップからのリストア手順は定期的に実際に実行して検証されなければならない (MUST)。目標復旧時間（RTO)と目標復旧時点（RPO）が定義され、検証結果と比較されなければならない (MUST)。

#### Scenario: リストア訓練
- **WHEN** 定期的なリストア検証を実施したとき
- **THEN** 復元完了までの時間とデータ損失範囲が計測され、RTO / RPO の達成状況が記録される

### Requirement: データのライフサイクル管理
オブジェクトストレージのデータには、アクセス頻度に応じたストレージクラス移行と保持期限のライフサイクルポリシーが定義されなければならない (SHALL)。個人情報等の機微データは保持要件に従って自動削除されなければならない (MUST)。

#### Scenario: 低頻度データの自動移行
- **WHEN** オブジェクトが定義された期間アクセスされなかったとき
- **THEN** 低コストのストレージクラスへ自動的に移行される
