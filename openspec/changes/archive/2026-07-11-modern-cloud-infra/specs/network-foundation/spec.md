# network-foundation

## ADDED Requirements

### Requirement: マルチAZ ネットワーク基盤
ネットワーク基盤（VPC 相当）は、少なくとも 2 つのアベイラビリティゾーンにまたがって構成されなければならない (MUST)。

#### Scenario: 単一 AZ 障害への耐性
- **WHEN** 1 つのアベイラビリティゾーンが利用不能になったとき
- **THEN** 残りの AZ のサブネットとルーティングでサービス提供が継続できる

### Requirement: サブネットの階層分離
サブネットはパブリック（インターネット向けロードバランサ等）、プライベート（アプリケーション）、データ（データストア）の 3 層に分離されなければならない (MUST)。アプリケーションとデータストアはプライベート以下の層に配置されなければならない (MUST)。

#### Scenario: データストアへの直接アクセス遮断
- **WHEN** インターネットからデータ層のリソースへ直接接続が試みられたとき
- **THEN** ルーティングおよびファイアウォールにより接続は到達しない

### Requirement: 明示的な通信制御
層間およびワークロード間の通信は、デフォルト拒否とし、必要な経路のみをセキュリティグループ／ネットワークポリシーで明示的に許可しなければならない (MUST)。

#### Scenario: 未許可通信の拒否
- **WHEN** 許可リストにない送信元からアプリケーション層のポートへ接続が試みられたとき
- **THEN** 接続は拒否され、フローログに記録される

### Requirement: 送信トラフィックの制御と記録
プライベートサブネットからのインターネット向け通信は NAT 経由とし、ネットワークフローログが有効化されなければならない (MUST)。マネージドサービスへの通信は可能な限りプライベートエンドポイントを使用しなければならない (SHALL)。

#### Scenario: フローログの取得
- **WHEN** ネットワーク上で通信が発生したとき
- **THEN** 送信元・宛先・許可/拒否の別がフローログとして保存され、調査に利用できる
